# Version Control Policies
Project Module: CS107.3 Object Oriented Programming with C#

## Branching Strategy
- `main`: Production-ready code used for final submission packaging.
- `development`: Feature integration branch where modules are tested together.

## Guidelines
- Always pull the latest changes before starting local work.
- Keep commit messages descriptive (e.g., use prefixes like 'Fix:', 'Feature:', 'Docs:').
- Ensure the project builds cleanly before running a push execution.
